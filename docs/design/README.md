# Design Documentation

Reference index for UX Insight's design documentation.

---

## Documents

### [PRODUCT_PRINCIPLES.md](../PRODUCT_PRINCIPLES.md)

Product decision framework. The "why" behind every design and feature decision — what to build, what to defer, and how to resolve tradeoffs. The authoritative source for product-level intent.

### [DESIGN_SYSTEM.md](DESIGN_SYSTEM.md)

Implementation reference for the design token layer. CSS custom property names and their light/dark values, typography scale, spacing scale, border radius, shadows, and component usage rules. Read this when adding or modifying components.

### [lovable-review.md](lovable-review.md)

Design analysis of the Lovable React prototype. Covers the OKLCH color palette, typography decisions, layout architecture, interaction patterns, dark mode strategy, and notes on what to carry forward into Angular. Read this when planning new views or major visual changes.

### [component-inventory.md](component-inventory.md)

Catalog of UI patterns found in the Lovable prototype. Covers primitives (Card, SeverityBadge, ScoreRing, Progress, Sparkline), the three-column app shell, dashboard components, report viewer sections, finding detail, and recurring micro-patterns. Use as a reference when implementing new components.
